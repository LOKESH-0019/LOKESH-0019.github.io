Lokesh Madanapalli Portfolio Contribution: Elder Ray Index & Crew AI Agent Stock Prediction

Financial markets are complex and driven by both price movements and investor sentiment. Traders often struggle to separate emotion from data when making decisions. The AI Agent Stock Prediction project addresses this by combining the Elder-Ray technical indicator with AI agents to provide objective investment recommendations.

The Elder-Ray Index quantifies Bull Power (high price minus moving average) and Bear Power (low price minus moving average), offering clear signals of market momentum. I designed and implemented the Elder-Ray calculation module, integrated it into a Streamlit interface supporting real-time and historical modes, and connected it with CrewAI agents to generate BUY, SELL, or HOLD recommendations. This solution ensures traders receive data-driven guidance with visual charts, automated decision tasks, and backtesting insights.

Project Overview & Sprint Objectives

The project was executed over four focused sprints, each delivering key functionality:

Sprint Objective
1 Minimal Viable Product – core Elder-Ray calculation and basic UI
2 Streamlit UI development & CrewAI integration for live recommendations
3 Unit & Integration Testing – ensure reliability and performance
4 Backtesting Framework – evaluate strategy on historical data

User Story & Task Summary

The primary user story: As a trader, I want a reliable AI-driven tool that analyzes market momentum using the Elder-Ray Index and delivers clear BUY/SELL/HOLD advice. Below is a concise overview of my key tasks; full details are in the appendix.

  • Sprint 1 (28 h): Elder-Ray module — wrote functions for EMA/SMA moving averages, Bull/Bear Power metrics, validated output against sample data.
  • Sprint 2 (22 h): UI & AI — developed Streamlit controls, added historical vs. real-time modes, integrated YahooFinance API and CrewAI task creation for decision support.
  • Sprint 3 (20 h): Testing & docs — implemented pytest suite covering 90% of code paths, created developer documentation and inline comments.
  • Sprint 4 (27 h): Backtesting & analysis — built backtesting engine, plotted performance metrics (Sharpe Ratio, drawdown), and generated summary reports.
